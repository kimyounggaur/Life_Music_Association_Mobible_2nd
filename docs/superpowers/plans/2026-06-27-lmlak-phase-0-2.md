# LMLAK Phase 0-2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first working static-export LMLAK mobile web app from the provided Markdown specs and `01 Source` assets.

**Architecture:** Use Next.js App Router with `output: "export"`, Tailwind tokens, local slugged assets, and client-safe data layers. Keep Supabase/Auth/Capacitor integration points isolated so later phases can replace dummy data without changing route or component contracts.

**Tech Stack:** Next.js, React, TypeScript, Tailwind CSS, Vitest, Supabase JS placeholders, Capacitor-ready static export.
