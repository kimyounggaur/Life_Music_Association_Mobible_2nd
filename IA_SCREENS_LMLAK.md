# 한국생활음악강사협회(LMLAK) 모바일앱 — 정보구조 & 화면 설계 (심화 / `IA_SCREENS_LMLAK`)

> **범위**: 본 문서는 마스터 사양서 `@BUILD_SPEC_LMLAK.md`의 **[정보구조 & 화면 설계]** 심화 문서다. 탭/라우트·스키마·인증·오디오 등 충돌 시 **마스터가 최종 기준**이다.
> **전제**: 기술 스택·아키텍처·인증·오디오 규칙은 첨부 `MOBILE_SPEC` 2종을 그대로 계승한다(Capacitor + Next.js App Router 정적 익스포트 + Supabase 직접 호출 + PKCE 딥링크).
> **참고 UX**: 오늘은뮤직(todaymusicgood.com)의 모듈형 카드 IA를 차용하되, '지점/가맹' → '자격증 과정·강사 지원·교육 콘텐츠·기관 매칭'으로 재구성한다.
> **정체성**: 우리는 **기관 수업(학교·복지관·문화센터·요양시설)에 특화된 강사 지원 협회**다. 콘텐츠의 원천은 `01 Source` 자산(과목 아이콘 20+종·연주자 일러스트·교재 9종·연습곡집 81종·소셜 로고).

## 0. 설계 원칙 요약 (이 섹션이 따르는 규칙)

| 원칙 | 적용 |
|---|---|
| 정적 익스포트 가능 형태 | 모든 라우트는 빌드타임에 결정. 동적 상세는 `[id]` + `generateStaticParams` **또는** 쿼리스트링/클라이언트 라우팅으로 처리. `middleware.ts`·서버 액션·`app/api/*` 사용 금지 |
| 보호 라우트 | 서버 가드 불가 → **클라이언트 라우트 가드**(`<AuthGuard>`)로 세션 체크 후 리디렉트 |
| 데이터 | Supabase JS SDK 직접 호출. RLS로 권한 통제(다운로드/강사라운지) |
| 자산 | 한글·공백 파일명 → 빌드 전 **영문 kebab-case 슬러그**로 리네이밍/매핑(섹션 7) |
| 없는 것 금지 | 시범영상·오디오 미존재 시 **placeholder**(자산 매핑 테이블의 `status`로 명시) |
| 터치 타깃 | 모든 인터랙티브 요소 44×44px+, 하단탭 56px 높이 |

### 1.1 동적 라우트 정적화 정책

| 라우트 | 처리 | 근거 |
|---|---|---|
| `/certifications/[subjectSlug]` | **generateStaticParams**로 20+종 전량 정적 생성 | 과목 수 고정·SEO/딥링크 유리 |
| `/contents/textbooks/[id]` | generateStaticParams(9종) | 수량 적음·고정 |
| `/contents/songbooks/[id]` | generateStaticParams(81종) | 수량 많지만 빌드타임 고정 |
| `/contents/demos/[id]`, `/contents/ppt/[id]` | generateStaticParams(빌드 시점 목록) | 콘텐츠는 빌드 시 스냅샷 |
| `/news/[id]`, `/reviews/[id]`, `/magazine/[id]` | generateStaticParams + **클라이언트 fetch 폴백** | 운영 중 신규 글은 재빌드 없이 `/news?id=` 클라이언트 라우팅으로 표시 |
| `/lounge/community/[id]`, `/lounge/jobs/[id]` | **클라이언트 라우팅(쿼리)** 우선 | 보호 + 빈번 갱신 → 정적 셸 + 런타임 fetch |

> **정적 익스포트 함정 점검(작업 전 필수)**: ① `middleware.ts` 금지 → `<AuthGuard>`. ② 서버 컴포넌트 런타임 로직 금지 → 클라이언트 컴포넌트 + Supabase SDK. ③ `app/api/*` 금지 → Edge Function 또는 직접 호출. ④ `next/image` → `unoptimized`. ⑤ 신규 콘텐츠 상세는 generateStaticParams가 빌드 시점만 커버하므로 **클라이언트 fetch 폴백** 라우트를 반드시 동반.

## 2. 글로벌 네비게이션 — 하단탭(Bottom Tab)

### 2.1 탭 구성과 근거

5개 탭. 엄지 한손 조작 도달 영역 안에 핵심 동선을 배치하고, 사용자 의도(탐색→학습→소속→개인)를 단계로 매핑한다.

| 순서 | 탭 | 라우트 | 역할 | 채택 근거 |
|---|---|---|---|---|
| 1 | **홈** | `/` | 진입·요약·CTA 허브 | 모든 진입의 기준점. 오늘은뮤직식 모듈 카드 대시보드 |
| 2 | **자격증과정** | `/certifications` | 과목 탐색→자격 신청(획득 동선) | 협회의 1차 가치=자격증. 가입 전환의 핵심 깔때기 |
| 3 | **교육콘텐츠** | `/contents` | 교재·연습곡집·PPT·시범연주 라이브러리 | 현장 R&D 산출물=차별점. 자산(81+9종)의 노출 무대 |
| 4 | **강사라운지** | `/lounge` | 강사 전용 지원(공지·자료·구인·커뮤니티) | '기관 수업 특화' 미션의 실현 공간. 강사 리텐션 핵심 |
| 5 | **마이** | `/me` | 개인(자격·찜·다운로드·설정) | 개인화·세션 의존 기능 집결 |

> **연주/실습(`/practice`)·검색(`/search`)은 탭이 아님.** 연주는 과목/콘텐츠 상세에서 진입(맥락 의존)하고 홈 그리드·콘텐츠 상세에 '바로 연습' 버튼으로 노출. 검색은 모든 탭 상단의 검색 아이콘(헤더)으로 전역 접근. 탭을 5개로 고정해 인지 부하·터치 정확도를 지킨다.

### 2.2 상태(헤더/탭) 추가 요소
- **상단 헤더(전 화면 공통)**: 좌측 엠블럼(홈 이동) · 중앙 화면 타이틀 · 우측 검색 아이콘 + 알림 벨(로그인 시 미읽음 배지).
- **하단탭 액티브 표시**: 아이콘+라벨, 활성 컬러 = Primary, 비활성 = Neutral-500. 세이프에어리어 하단 인셋만큼 패딩.

### 2.3 권한별 노출 차이

| 탭/요소 | 비로그인 | 로그인(일반회원) | 강사(승인됨) | 관리자 |
|---|---|---|---|---|
| 홈 | ✅ | ✅ | ✅ | ✅ |
| 자격증과정 | ✅ | ✅ | ✅ | ✅ |
| 교육콘텐츠 | ✅(미리보기) | ✅(미리보기) | ✅+**다운로드** | ✅+다운로드 |
| 강사라운지 | ✅ 탭은 보임 → 탭하면 **안내+강사신청 유도 화면** | 동일(미강사) | ✅ **전체 기능** | ✅ |
| 마이 | ✅ 탭은 보임 → 탭하면 **로그인 유도** | ✅ | ✅(+강사 메뉴) | ✅ |
| 헤더 알림 벨 | ❌ 숨김 | ✅ | ✅ | ✅ |
| 콘텐츠 다운로드 버튼 | "강사 전용" 잠금 표시 | "강사 전용" 잠금 표시 | ✅ 활성 | ✅ |
| 관리자(`/admin`) | 차단 | 차단 | 차단 | ✅(관리자 가드) |
| 로그인/로그아웃 CTA | 로그인 노출 | 로그아웃 노출 | 로그아웃 노출 | 로그아웃 노출 |

> **설계 결정**: 강사라운지·마이 탭은 비로그인에게도 **탭 자체는 노출**(존재를 알려 가입 동기 부여)하되, 진입 시 권한 게이트 화면으로 전환한다. 탭을 숨기면 강사 가치 제안이 보이지 않아 전환에 불리.

## 3. 화면별 상세 명세

> 각 화면 공통 항목: **목적 / 주요 컴포넌트 / 데이터 소스 / 상태(빈·로딩·에러) / 주요 인터랙션 / 보호 여부**.
> 로딩=스켈레톤(카드/리스트 형태 유지), 빈=일러스트+안내+CTA, 에러=메시지+[재시도]+오프라인 분기.

### 3.1 홈 `/` 🔓
- **목적**: 협회 가치 한눈 요약 + 핵심 동선(자격·콘텐츠·연습) 분기 + 회원/강사 등록 전환.
- **주요 컴포넌트**:
  1. 히어로(엠블럼+슬로건 "현장에서 자란 음악교육, 기관 수업의 표준") — 그라데이션 배경.
  2. 협회 한줄소개 카드(미션 요약 + [협회소개 더보기]→`/about`).
  3. **과목 바로가기 그리드**(과목 아이콘 4×n, 탭→`/certifications/[slug]`).
  4. 추천 교육콘텐츠 캐러셀(연습곡집/교재 표지 카드).
  5. 공지/뉴스 미니리스트(최신 3건→`/news`).
  6. 최근 연습곡집 가로 스크롤(표지 썸네일→상세).
  7. **CTA 듀얼 버튼**: [회원가입] / [강사 등록 안내].
- **데이터 소스**: `subjects`(과목), `contents`(추천 플래그), `posts`(news), Supabase Storage(표지). 비로그인도 전부 공개 데이터.
- **상태**: 로딩=섹션별 스켈레톤. 빈(콘텐츠 0)=추천 캐러셀 숨기고 자격증 그리드만. 에러=섹션 단위 폴백(한 섹션 실패해도 나머지 렌더, 실패 섹션에 [다시 불러오기]).
- **인터랙션**: 그리드 탭, 캐러셀 스와이프, CTA 탭, pull-to-refresh.
- **보호**: 🔓.

### 3.2 협회소개 `/about` 🔓 (+ 하위 6)
- **목적**: 브랜드 신뢰 형성. '기관 수업 특화' 정체성·교육철학·운영진 전달.
- **주요 컴포넌트**: 섹션 인덱스(스토리/미션/철학/교육이사/연혁/아이덴티티 앵커), 각 섹션 카드.
  - story: 협회 설립 배경 내러티브.
  - mission: 미션 원문("기관 수업에 특화…현장 R&D") + 핵심 가치 4뱃지.
  - philosophy: 교육철학 항목 리스트.
  - directors: 협회장·교육이사 카드(사진·직함·담당 과목). *사진 없으면 엠블럼 placeholder.*
  - history: 연혁 타임라인(연도-사건).
  - identity: 컬러 팔레트 칩 + 엠블럼(`협회소개.jpg`/엠블럼) + 영문 풀네임.

- **상태**: 로딩="인증 처리 중". 에러="링크가 만료되었거나 잘못되었습니다"+[다시 로그인]. 성공(웹)=홈 리디렉트.
- **딥링크 핸들러(라우트 아님)**: 네이티브 `@capacitor/app` `appUrlOpen` 리스너가 `com.lmlak://auth-callback` 수신→`exchangeCodeForSession`→온보딩/홈.
- **인터랙션**: [앱에서 열기] 탭(사용자 제스처로 custom scheme 트리거).
- **보호**: 🔓.
- **주의**: 이메일 템플릿엔 https만(custom scheme 직접 삽입 시 `#ZgotmplZ` 깨짐). `com.lmlak://*`는 Supabase Additional Redirect URLs에 와일드카드 등록.

- **보호**: 🔒(첫 로그인 1회).

### 3.13 강사 등록 신청 `/auth/instructor-apply` 🔒
- **목적**: 일반회원→강사 권한 신청(자격 보유/응시 정보 제출).
- **컴포넌트**: 신청 폼(이름·연락처·보유 자격·신청 과목·증빙 첨부), 약관, [제출]. 제출 후 "검토 중" 상태 카드.
- **데이터 소스**: `instructor_applications`(status: pending/approved/rejected).
- **상태**: 로딩=폼 비활성+스피너. 빈=N/A. 에러=검증 메시지/제출 실패 토스트.
- **인터랙션**: 첨부 업로드, 제출, 상태 확인.
- **보호**: 🔒.

### 3.14 검색 `/search` 🔓
- **목적**: 과목·콘텐츠·뉴스 통합 검색.
- **컴포넌트**: 검색 입력(자동 포커스), 최근/인기 검색어, 결과 탭(전체/과목/콘텐츠/소식), 결과 카드.
- **데이터 소스**: Supabase 텍스트 검색(`ilike`/FTS). 로컬 최근검색.
- **상태**: 로딩=결과 스켈레톤. 빈(0건)="검색 결과가 없습니다"+추천 키워드. 에러=[재시도].
- **인터랙션**: 입력 디바운스, 탭 전환, 결과 탭→상세, 검색어 삭제.
- **보호**: 🔓.

### 3.15 설정 `/me/settings` 🔒
- (3.9에 포함. 언어/캐시/약관/버전/라이선스.) 다국어 토글은 i18n 적용 후 노출(섹션 5 참고).

### 3.16 오프라인/네트워크 에러 (전역) 🔓
- **목적**: 단절 시에도 앱이 죽지 않게 명확한 회복 동선 제공.
- **컴포넌트**: 전역 네트워크 배너(상단 슬라이드인 "오프라인 상태입니다"), 화면 단위 에러 상태(일러스트+[재시도]), 캐시된 콘텐츠(오프라인 학습자료) 접근 안내.
- **데이터 소스**: `@capacitor/network` 상태, 캐시(Storage/IndexedDB) 폴백.
- **상태**: 오프라인=배너+재시도. 일부 캐시 가능=캐시 콘텐츠 노출+"오프라인 모드". 복구 시 자동 재요청.
- **인터랙션**: [재시도], 캐시 콘텐츠 열기.
- **보호**: 🔓. (오프라인 학습자료 = Apple '단순 웹뷰' 거부 방어의 네이티브 가치.)

### 3.17 404 / 관리자 `/admin` 🔒🛠
- **404**: 잘못된 경로→일러스트+[홈으로]. 🔓.
- **관리자**: 콘텐츠/회원/자격 신청 승인 대시보드. **관리자 전용 가드**(`role='admin'`). 비관리자 접근 차단→홈. 🔒🛠.

## 4. 핵심 사용자 플로우 (텍스트 플로우차트)

### ① 소셜 로그인 → 세션 영속
```
[로그인] 소셜버튼 탭(Google/Apple/Naver/Kakao)
  → signInWithOAuth({ skipBrowserRedirect:true })  (PKCE)
  → @capacitor/browser Browser.open(authUrl)
  → 사용자 제공자 화면에서 인증
  → 제공자 → https://(도메인)/auth/callback  (브리지)
  → 사용자 [앱에서 열기] 탭 → com.lmlak://auth-callback?code=...
  → @capacitor/app appUrlOpen 리스너 캐치
  → supabase.auth.exchangeCodeForSession(code)
  → 세션 localStorage 영속(persistSession, autoRefreshToken)
  → 신규? → /auth/onboarding : 홈
  ──(앱 재실행)── 저장 세션 복원 → 로그인 유지 ✓
  [에러] 취소→로그인 화면 / 만료→"다시 로그인"
```

### ② 매직링크 메일 → 앱 복귀
```
[로그인] 이메일 입력 → [매직링크 보내기]
  → signInWithOtp(email, { emailRedirectTo: https://(도메인)/auth/callback })
  → "메일 확인" 안내 화면
  → (메일) 사용자가 링크(https만) 탭 → /auth/callback 브리지
  → [앱에서 열기] → com.lmlak://auth-callback?code=...
  → appUrlOpen → exchangeCodeForSession → 세션 영속 → 홈/온보딩
  [방어] 이메일 스캐너 토큰 조기소모·PKCE 끊김 → https 브리지가 흡수
  [에러] 만료/이미사용 → "링크 만료, 다시 요청"
```

### ③ 과목 탐색 → 자격 신청
```
[홈] 과목 그리드 탭  또는  [자격증과정] 그리드
  → /certifications/[subjectSlug] (소개·커리큘럼·시험요건·연계교재)
  → [자격 신청하기] 탭
  → 로그인? ──아니오──> 로그인 시트 → (플로우①/②) → 복귀
        └──예──> /auth/instructor-apply?subject=[slug]
  → 폼 작성·증빙 첨부 → 제출 → instructor_applications(pending)
  → "검토 중" 상태 → (관리자 승인) → role=instructor → 강사라운지 해금
```

### ④ 연습곡집 → 백킹 재생
```
[교육콘텐츠] 세그먼트=연습곡집 → 필터(악기/대상/난이도/키)
  → 곡집 카드 탭 → /contents/songbooks/[id]
  → 미리보기 확인 → 오디오 백킹 플레이어 [재생]
  → 첫 제스처에서 AudioContext.resume() (iOS 무음=playback 세션)
  → 재생/일시정지/스크럽
  [강사면] [다운로드] 가능 (플로우⑤)
  [에러] 재생 실패 → 인라인 "다시 시도" / 오프라인 → 캐시 폴백
```

### ⑤ 강사 자료 다운로드 (권한 체크)
```
[콘텐츠 상세/라운지 자료실] [다운로드] 탭
  → 세션 있나? ──없음──> 로그인 시트 → 복귀
  → role=instructor? (RLS + 클라이언트 사전 체크)
        ──아니오──> "강사 전용 콘텐츠입니다" + [강사 등록 안내]
        ──예──> Supabase 서명 URL 발급(RLS 통과) → 파일 다운로드
  → @capacitor/filesystem 저장(오프라인 학습자료) → downloads 기록
  [에러] RLS 거부→권한 안내 / 네트워크→재시도
```

### ⑥ 보호 라우트 미로그인 차단 → 로그인 유도
```
사용자가 /lounge/* 또는 /me/* 진입(탭/딥링크)
  → <AuthGuard> 세션 체크(클라이언트, 미들웨어 대체)
  → 세션 없음 ──> 현재 경로 저장(returnTo) → /auth/login 리디렉트
  → 로그인 성공(플로우①/②) → returnTo로 복귀
  → 세션 있으나 권한 부족(/lounge=강사 아님) ──> 게이트 화면(강사 신청 유도)
  → 관리자 라우트(/admin)는 role=admin 확인, 아니면 홈
```
